# ETOH Training System v2 — AMC 10 → AIME Qualification

## Context
- **Student:** 13 years old, 8th grade
- **Current level:** Halfway through AoPS Intro to Algebra. Starting AoPS Counting & Probability March 22, 2026.
- **Goal:** AMC 10 in November 2026, AIME qualifying score (115-120+ target, cutoff typically ~103-106)
- **Timeline:** ~8 months. Not casual study — full commitment.

---

## The ETOH Difficulty Scale

| Rating | Description |
|--------|-------------|
| 1-3 | Mechanical, single technique |
| 4-6 | Requires layering 2 ideas |
| 7-9 | Non-obvious, intro AIME territory |
| 10-12 | Hidden tier — AIME/USAJMO/IMO problems |

Self-assigned after every problem. The act of rating forces reflection on *why* it was hard, not just whether it was solved. Calibrate periodically against AoPS/MAA difficulty data — if a problem was statistically easy for the testing cohort but felt like a 7, that flags a specific gap, not a general wall.

---

## Error Taxonomy (Tag Every Miss)

Every wrong answer or wall gets tagged with one of three types:

| Tag | Meaning | Fix |
|-----|---------|-----|
| **K** | Knowledge gap — didn't know the tool | Go back to relevant AoPS chapter |
| **P** | Pattern recognition gap — knew the tool, didn't see it | More trigger sentence reps, layering problems |
| **E** | Execution error — arithmetic/careless/computation | Slow down, check work protocol |

Track which tag dominates. If K is everywhere, you need more book time. If P dominates, you need more costumed-technique problems. If E dominates, you have a computation hygiene issue, not a math issue. These are different problems requiring different training.

## Phase 2 — AMC 10 Tiered Grind
Pull last 10 AMC 10 tests (25 problems each — note: 25, not 20).

**Protocol:**
- Start with problems 1-10 across all tests
- Solve cold, no hints
- After each problem log:
  - ETOH rating
  - Error tag (K / P / E) if wrong or walled
  - Trigger sentence: *"I knew to [X] because I saw [Y]"*
  - If wrong: *"I should have [X] because [Y] was there"*
  - Second sentence: *"What else looked tempting, and why was it wrong?"*

Speed matters. Problems 1-10 should feel fast once foundation is solid.

---

## Phase 3 — Progressive Difficulty
Move to problems 11-25 across the same tests. This is where AIME qualification is won or lost.

**Protocol:**
- Time each problem
- If no new structure after 10-15 min: log as **wall**, look at solution, move on
  - If you're actively making progress, keep going
  - If you're looping the same idea, stop immediately — time doesn't matter, spinning does
- **Wall log:** problem + reason + error tag

**Weekly wall review:**
- Re-attempt every wall from the past week cold
- Solved → learned
- Persists after 2 weeks → go back to the AoPS chapter, not just the problem

---

## Timed Full Tests (Starting September)
AMC 10 is 25 problems in 75 minutes — 3 minutes per problem average. Solving correctly untimed and solving under pressure are different skills.

**Protocol from September onward:**
- Full 75-minute timed test every 2 weeks minimum
- Practice: when to skip, when to guess, how to allocate time
- Log: which problems you skipped, which you got wrong under time that you knew cold

Pacing strategy is a separate skill from problem-solving. Train it explicitly.

---

## Retention Checks (Integrated Throughout)
Every 5 new problems solved, pull 1 random old problem cold — no notes.

| Result | Meaning |
|--------|---------|
| Faster than before | Genuinely learned |
| Same speed | Retained but not internalized |
| Slower or stuck | Needs re-review, flagged |

---

## Core Daily Habit
After **every** problem:
1. Trigger sentence: *"I knew to [X] because I saw [Y]"*
2. If wrong/walled: error tag (K / P / E) + what should have told you
3. Temptation sentence: *"[Z] looked right because [W], but it was wrong because [V]"*

The temptation sentence is what trains fast elimination of wrong paths — critical for AMC time management.

## What This System Does Not Include (By Design)
- **AMC 12 phase** — removed entirely. AMC 12 problems 15-25 require trig, logs, complex numbers outside the AMC 10 syllabus. Mastering the AMC 10 ceiling first is the right call.
- **AIME-specific training** — that's the next cycle if November goes well. Don't split focus now.

---

## Mastery Definition
A topic is stable when you can:
- Solve standard versions quickly
- Recognize the pattern in mixed-topic sets (not just chapter-blocked practice)
- Explain the trigger cleanly in one sentence
- Not repeat the same error tag on the same topic twice in a row
- Revisit an old problem later and solve it faster

Without this definition, you cover material without knowing what stuck.
