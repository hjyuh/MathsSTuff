# Research-Augmented Training

## The Breakthrough Pattern

What happened with Erdős problems:
```
1. AI researches vast space of potential approaches
2. Identify promising paths humans might miss
3. Verify partial progress rigorously
4. Combine techniques from different domains
5. Formal proof generation
```

## Applying to IMO Training

### Standard Approach
```
See problem → Try approaches → Stuck or solve
```

### Research-Augmented Approach
```
See problem
     ↓
AI maps solution landscape:
├── Similar problems in database
├── Promising techniques (ranked)
├── Common failure modes
├── Unexpected connections
└── Solution structure prediction
     ↓
You execute with full context
     ↓
If stuck, you have backup paths mapped
```

## The Research Protocol (For Hard Problems)

```markdown
## Deep Research Protocol

### Phase 1: Classification (5 min)
- Problem type, subjects, precise ask

### Phase 2: Database Search (10 min)
AI finds 10 most similar problems, explains connections

### Phase 3: Technique Landscape (10 min)
AI maps all plausible approaches with reasoning

### Phase 4: Failure Analysis (5 min)
Common errors, partial solution traps

### Phase 5: Structure Prediction (5 min)
What clean solution probably looks like

### Phase 6: Execute (You)
Solve with full context

### Phase 7: Compare and Extract (15 min)
Compare to 7/7 solutions, Lean verification, update banks
```

## The 7/7 Solution Pipeline

```
Human 7/7 solution (from contestant)
              ↓
        [Aristotle]
              ↓
Lean proof (every step verified)
              ↓
    [Claude translation]
              ↓
Natural language with ZERO gaps
              ↓
  [Compare to your attempt]
              ↓
See exactly where your proof had holes
```

## Problem Creation Path

**Stage 1: Identify Generators**
- Every problem has underlying structure
- Label the "generator" for each problem you solve

**Stage 2: Create Variations**
- Take solved problem, modify parameters
- Same technique, different surface

**Stage 3: Add Complexity Layers**
- Base technique + constraint + embedding = harder problem

**Stage 4: Cross-Pollinate**
- Combine techniques from different subjects
- Novel problems that require synthesis

**Goal:** Submit problems to competitions (USAMTS, regionals)

## Training vs. Competition

| Training (unlimited AI) | Competition (no AI) |
|------------------------|---------------------|
| Full research on every problem | Drawing on trained intuition |
| Explicit landscape mapping | Implicit pattern recognition |
| AI builds your understanding | Brain has internalized patterns |

AI accelerates training. Competition tests what you've internalized.

## IMO-Level Progression

- Shift focus to IMO Shortlist
- Study 7/7 solutions from actual contestants
- Full Lean pipeline for verification
- Problem creation at IMO level
